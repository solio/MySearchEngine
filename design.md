# AI搜索MCP 选型与设计方案

## TODO 1: 自托管AI友好搜索服务选型

### 可选方案对比

| 方案 | 类型 | 部署难度 | AI友好度 | 成本 | 推荐度 |
|------|------|----------|----------|------|--------|
| ** SearXNG ** | 元搜索引擎 | 中 | 高 | 免费 | ⭐⭐⭐⭐⭐ |
| ** Meilisearch ** | 搜索引擎 | 低 | 中 | 免费 | ⭐⭐⭐ |
| ** Typesense ** | 搜索引擎 | 低 | 中 | 免费 | ⭐⭐⭐ |
| ** LocalGPT + Search ** | LLM + 搜索 | 高 | 很高 | 免费 | ⭐⭐⭐⭐ |
| ** Brave Search API ** | 商业API | 低 | 高 | 付费 | ⭐⭐⭐ |

### 推荐方案：SearXNG

**核心优势：**
- 完全开源、自托管，无调用次数限制
- 元搜索：聚合多个搜索引擎结果（Google、Bing、DuckDuckGo等）
- 内置AI友好的内容提取功能
- Docker一键部署，支持Mac/WSL
- 可配置搜索源、权重、过滤规则
- RESTful API，易于集成

**部署方式：**
```bash
# Docker部署
docker run -d -p 8080:8080 searxng/searxng
```

**API使用示例：**
```
GET http://localhost:8080/search?q=隆基绿能&format=json
```

### 备选方案组合

**SearXNG + 自定义后处理层：**
- SearXNG负责搜索聚合
- 自定义服务负责内容提取、去重、质量评分

## TODO 2: 精品站点/URL模式筛选模型设计

### 核心思路

采用**混合规则+机器学习**的渐进式方案：

### Phase 1: 规则引擎（快速上线）

#### 垃圾信息识别规则
```python
垃圾关键词模式 = [
    r"实时行情", r"分时走势", r"K线图", r"技术分析",
    r"大盘指数", r"股吧首页", r"F10资料", r"财务指标",
    r"盘口分析", r"资金流向", r"龙虎榜"
]

优质站点白名单 = [
    "xueqiu.com",       # 雪球
    "guba.eastmoney.com", # 股吧（需进一步筛选帖子）
    "36kr.com",         # 36氪
    "yicai.com",        # 第一财经
    "caixin.com",       # 财新
    "cls.cn",           # 财联社
    "stcn.com",         # 证券时报
    "cs.com.cn",        # 中国证券报
    "nbd.com.cn",       # 每日经济新闻
    "zhihu.com",        # 知乎（需筛选）
    "weibo.com",        # 微博（需筛选）
]

优质URL模式 = [
    r"/\d{6,8}/",  # 包含日期的文章URL
    r"/news/",
    r"/article/",
    r"/post/",
    r"/p/\d+",     # 知乎/雪球帖子
]

垃圾URL模式 = [
    r"/list,",     # 股吧列表页
    r"/quote/",    # 行情页
    r"/kline/",    # K线页
    r"/chart/",    # 图表页
]
```

#### 评分机制
```
初始分数 = 100

# 站点加分
如果站点在白名单: +50
如果是已知优质域名: +30

# URL模式加分
如果匹配优质URL模式: +20

# 内容减分
每匹配一个垃圾关键词: -10
如果垃圾关键词超过3个: 直接过滤

# 标题长度加分
标题长度在20-80字: +10

# 最终
分数 > 60: 保留
分数 > 90: 精品
```

### Phase 2: 机器学习模型（持续优化）

#### 训练数据收集
```json
{
  "url": "https://...",
  "title": "...",
  "content": "...",
  "domain": "...",
  "label": 1,  // 1=精品, 0=普通, -1=垃圾
  "user_feedback": null  // 用户后续可以反馈
}
```

#### 特征工程
| 特征类型 | 特征 | 说明 |
|----------|------|------|
| URL特征 | 域名 | one-hot或embedding |
| | 路径模式 | 正则匹配特征 |
| | 长度 | 数值 |
| 标题特征 | 长度 | 数值 |
| | 关键词匹配 | 垃圾/优质关键词计数 |
| | 情感倾向 | 简单正负向 |
| 内容特征 | 垃圾关键词密度 | 数值 |
| | 技术术语密度 | 个股/行情词占比 |
| | 内容长度 | 数值 |
| | 链接数量 | 数值 |
| 历史特征 | 该域名历史精品率 | 数值 |
| | 该URL模式历史精品率 | 数值 |

#### 模型选择
- **轻量级方案**：Logistic Regression + TF-IDF
- **中量级方案**：XGBoost/LightGBM + 人工特征
- **重量级方案**：小BERT（如DistilBERT）微调

#### 反馈循环
```
用户标记(好/差) → 更新训练数据 → 定期 retrain → 模型优化
```

## TODO 3: 定向搜索设计

### 方案：site: 指令 + 多轮搜索

### 定向搜索API设计

```python
async def targeted_search(
    query: str,
    sites: List[str] = None,  # 优先搜索这些站点
    site_weight_overrides: Dict[str, float] = None,  # 站点权重
    require_all_sites: bool = False,  # 是否每个站点都要有结果
    max_results_per_site: int = 5,
) -> List[SearchResult]:
    pass
```

### 搜索策略

**策略1: 直接site搜索**
```
query = "隆基绿能 2024年报"
sites = ["xueqiu.com", "cls.cn", "36kr.com"]

# 生成多个搜索查询
searches = [
    f"site:xueqiu.com {query}",
    f"site:cls.cn {query}",
    f"site:36kr.com {query}",
    query  # 兜底搜索
]
```

**策略2: 混合搜索 + 后过滤重排**
```
1. 先用普通搜索获取结果
2. 按站点分组
3. 白名单站点结果前置
4. 非白名单站点结果后置或过滤
```

### 优质站点配置

```yaml
# config/quality_sites.yaml
```
