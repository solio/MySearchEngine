# 20260430 - 搜索调优迭代一思考过程

## 本次更新内容
1. 在规则引擎中添加详细评估参数输出
2. 添加--debug选项，将详细评估写入搜索结果
3. 显示问题识别统计（垃圾信息数、原因等）
4. 更新SearchResult模型存储评估详情
5. 优化main.py输出评估过程

## 思考过程

### 1. 需要输出哪些评估参数
- 域名是否在白名单
- URL模式匹配情况（好/坏）
- 垃圾关键词匹配数
- 标题长度是否合适
- 每一项的加分/减分详情
- 最终得分计算过程

### 2. 如何存储评估详情
在SearchResult中添加一个debug_info字典，保存：
- 各评估项的原始值
- 加分/减分
- 判断理由

### 3. Debug选项实现
- 命令行参数--debug
- 开启后在保存的markdown中显示详细评估
- 控制台也输出评估过程

### 4. 问题识别
- 统计过滤了多少垃圾
- 列出垃圾原因分类
- 提示是否需要更新参数（比如垃圾关键词过多）
